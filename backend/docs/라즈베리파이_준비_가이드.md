## 3. EC2 서버에서 실행되는 것들

`docker-compose.yml` 실행 시 4개 컨테이너가 생성됩니다:

| 컨테이너 | 역할 | 포트 |
|----------|------|------|
| `autobox-mqtt` | MQTT 브로커 (Mosquitto) | 8883 (외부), 1883 (내부) |
| `autobox-backend` | API 서버 (FastAPI) | 8000 |
| `autobox-frontend` | 웹 UI (Vue.js) | 80 |
| `autobox-mysql` | 데이터베이스 (선택) | 3306 |

**실행 명령:**
```bash
# EC2 서버에서
cd /home/ubuntu/git/S14P11A403/backend
docker-compose up -d --build
```

## 4. 라즈베리파이에서 준비할 것

### 4.1 필요한 파일 (3가지)

| 파일 | 위치 | 설명 |
|------|------|------|
| `mqtt_client.py` | `/home/pi/autobox/` | MQTT 클라이언트 코드 |
| `.env` | `/home/pi/autobox/` | 환경변수 설정 |
| `ca.crt` | `/home/pi/autobox/certs/` | EC2 서버 인증서 |

### 4.2 폴더 구조

```
/home/pi/autobox/
├── mqtt_client.py      ← EC2와 MQTT 통신하는 메인 파일
├── .env                ← 비밀번호, 서버 IP 등 설정
├── requirements.txt    ← Python 패키지 목록
└── certs/
    └── ca.crt          ← EC2 MQTT 브로커 인증서
```

## 5. 라즈베리파이 설정 (단계별)

### Step 1: 폴더 생성

```bash
# 라즈베리파이 터미널에서
mkdir -p /home/pi/autobox/certs
cd /home/pi/autobox
```

### Step 2: 파일 복사 (EC2 → 라즈베리파이)

**방법 A: USB로 복사**
1. EC2 서버에서 파일 다운로드
2. USB에 저장
3. 라즈베리파이에 복사

**방법 B: SCP로 직접 전송 (EC2에서 실행)**
```bash
# EC2 서버에서 실행
cd /home/ubuntu/git/S14P11A403/backend

# 인증서 복사
scp mqtt/certs/ca.crt pi@라즈베리파이IP:/home/pi/autobox/certs/

# 클라이언트 코드 복사
scp raspberry-pi/mqtt_client.py pi@라즈베리파이IP:/home/pi/autobox/
scp raspberry-pi/requirements.txt pi@라즈베리파이IP:/home/pi/autobox/
scp raspberry-pi/.env.example pi@라즈베리파이IP:/home/pi/autobox/.env
```

### Step 3: 환경변수 설정

```bash
# 라즈베리파이에서
cd /home/pi/autobox
nano .env
```

**수정할 내용:**
```env
# EC2 서버 IP (실제 IP로 변경!)
EC2_BROKER_HOST=43.201.254.235
EC2_BROKER_PORT=8883

# MQTT 인증 (EC2에서 설정한 비밀번호)
MQTT_USERNAME=raspberry-pi
MQTT_PASSWORD=여기에_실제_비밀번호_입력

# 인증서 경로
CA_CERT_PATH=/home/pi/autobox/certs/ca.crt

# MQTT 설정
TOPIC_PREFIX=autobox
CLIENT_ID=raspberry-pi-client
```

### Step 4: Python 패키지 설치

```bash
cd /home/pi/autobox
pip3 install paho-mqtt python-dotenv
```

또는
```bash
pip3 install -r requirements.txt
```

### Step 5: 인증서 권한 설정

```bash
chmod 644 /home/pi/autobox/certs/ca.crt
```

### Step 6: 실행 테스트

```bash
cd /home/pi/autobox
python3 mqtt_client.py
```

**정상 실행 시 출력:**
```

  - autobox/command/#
```

## 6. 명령 수신 테스트

### 6.1 프론트엔드에서 테스트 (웹 브라우저)

1. 웹 브라우저에서 `http://EC2서버IP` 접속
2. 박스 개수 입력 → 전송 버튼 클릭
3. 라즈베리파이 터미널에서 수신 확인

### 6.2 curl로 테스트 (터미널)

```bash
# 아무 컴퓨터에서 실행

  -H "Content-Type: application/json" \
```

## 7. 자동 실행 설정 (선택)

라즈베리파이 부팅 시 자동으로 mqtt_client.py 실행:

```bash
# 서비스 파일 생성
sudo nano /etc/systemd/system/autobox.service
```

**내용:**
```ini
[Unit]
Description=AutoBox MQTT Client
After=network.target

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi/autobox
ExecStart=/usr/bin/python3 /home/pi/autobox/mqtt_client.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

**활성화:**
```bash
sudo systemctl daemon-reload
sudo systemctl enable autobox
sudo systemctl start autobox

# 상태 확인
sudo systemctl status autobox
```

## 8. 문제 해결

### 연결 안 됨

| 증상 | 원인 | 해결 |
|------|------|------|
| `Connection refused` | EC2 보안그룹 | AWS 콘솔에서 8883 포트 열기 |
| `Authentication failed` | 비밀번호 오류 | `.env` 파일의 MQTT_PASSWORD 확인 |
| `Certificate error` | 인증서 문제 | ca.crt 파일 존재 및 경로 확인 |
| `Timeout` | 네트워크 문제 | 인터넷 연결 확인 |

### 디버깅 명령

```bash
# EC2 서버 연결 테스트
ping EC2서버IP

# MQTT 포트 연결 테스트
nc -zv EC2서버IP 8883

# 인증서 확인
openssl x509 -in /home/pi/autobox/certs/ca.crt -text -noout
```
