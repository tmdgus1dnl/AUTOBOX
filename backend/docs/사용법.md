## 1. 프로젝트 구조

```
backend/
├── docker-compose.yml    ← 메인 실행 파일
├── .env.example          ← 환경변수 템플릿
├── .env                  ← 실제 환경변수 (git 무시)
├── Dockerfile            ← FastAPI 빌드
├── app/                  ← FastAPI 애플리케이션
│   └── services/
│       └── mqtt.py       ← MQTT 서비스

│   ├── config/
│   │   ├── mosquitto.conf
│   │   ├── acl           ← 접근 제어
│   │   └── passwd        ← 비밀번호 (생성 필요)
│   └── certs/            ← TLS 인증서 (생성 필요)
```

## 2. EC2 서버 설정

### 2.1 사전 준비

```bash
# EC2 접속
ssh -i your-key.pem ubuntu@<EC2_PUBLIC_IP>

# Docker 설치 확인
docker --version
docker-compose --version

# 프로젝트 클론
git clone <REPO_URL>
cd S14P11A403/backend
```

### 2.2 인증서 생성

```bash
cd mqtt/certs
chmod +x generate-certs.sh
./generate-certs.sh <EC2_PUBLIC_IP>

# 예시
./generate-certs.sh 43.201.254.235
```

**생성되는 파일:**

| 파일 | 용도 |
|------|------|
| `ca.crt`, `ca.key` | CA 인증서 |
| `server.crt`, `server.key` | EC2 브로커용 |
| `client.crt`, `client.key` | 라즈베리파이용 |

### 2.3 비밀번호 생성

```bash
cd ../config
chmod +x generate-passwd.sh
./generate-passwd.sh
```

3개 사용자 비밀번호 설정:

| 사용자 | 용도 |
|--------|------|
| `backend` | FastAPI 서버 |
| `raspberry-pi` | 라즈베리파이 |
| `dashboard` | 대시보드 (읽기 전용) |

### 2.4 환경변수 설정

```bash
cd ../..  # backend/ 폴더로 이동
cp .env.example .env
nano .env
```

**필수 수정 항목:**

```env
# 데이터베이스 (Azure MySQL 등)
DATABASE_URL=mysql+pymysql://user:password@host:3306/autobox

# MQTT 비밀번호 (2.3에서 설정한 backend 비밀번호)
MQTT_PASSWORD=your_backend_mqtt_password

# 프론트엔드 API URL (EC2 IP로 변경)
VITE_API_URL=http://<EC2_PUBLIC_IP>:8000/api/v1

# CORS (EC2 IP 추가)
CORS_ORIGINS=http://localhost,http://<EC2_PUBLIC_IP>
```

### 2.5 AWS 보안그룹 설정

| 포트 | 프로토콜 | 소스 | 용도 |
|------|----------|------|------|
| 22 | TCP | My IP | SSH |
| 80 | TCP | 0.0.0.0/0 | Frontend |
| 8000 | TCP | 0.0.0.0/0 | Backend API |
| 8883 | TCP | 0.0.0.0/0 | MQTT TLS |
| 1883 | TCP | 127.0.0.1 | MQTT 내부 (선택) |

### 2.6 Docker 실행

```bash
# 전체 스택 실행 (MQTT + Backend + Frontend)
docker-compose up -d --build

# 로그 확인
docker-compose logs -f

# 개별 서비스 로그
docker logs -f autobox-mqtt
docker logs -f autobox-backend
docker logs -f autobox-frontend
```

### 2.7 실행 옵션

```bash
# 기본 실행 (외부 DB 사용)
docker-compose up -d --build

# MySQL 포함 실행 (로컬 개발용)
docker-compose --profile with-db up -d --build

# 특정 서비스만 실행
docker-compose up -d mosquitto backend
```

## 3. 라즈베리파이 연결

### 3.1 아키텍처

```
┌──────────────┐      로컬 MQTT       ┌──────────────────────────────┐
│  오린 나노   │ ◀──────────────────▶ │        라즈베리파이          │
│ (자율주행차) │     localhost:1883   │                              │
└──────────────┘                      │  ┌────────────────────────┐  │
                                      │  │ 로컬 Mosquitto 브로커  │  │
                                      │  │      (Port 1883)       │  │
                                      │  └───────────┬────────────┘  │
                                      │              │               │
                                      │              ▼               │
                                      │  ┌────────────────────────┐  │
                                      │  │     Bridge Module      │  │
                                      │  │  (EC2로 메시지 전달)   │  │
                                      │  └───────────┬────────────┘  │
                                      └──────────────┼───────────────┘
                                                     │
                                        TLS 8883 (Outbound)
                                                     │
                                                     ▼
                                      ┌──────────────────────────────┐
                                      │     EC2 MQTT Broker          │
                                      │    <EC2_IP>:8883             │
                                      └──────────────────────────────┘
```

### 3.2 인증서 복사 (EC2 → 라즈베리파이)

```bash
# EC2에서 실행
scp mqtt/certs/ca.crt pi@<RPI_IP>:/home/pi/autobox/certs/
scp mqtt/certs/client.crt pi@<RPI_IP>:/home/pi/autobox/certs/
scp mqtt/certs/client.key pi@<RPI_IP>:/home/pi/autobox/certs/
```

### 3.3 라즈베리파이 초기 설정

```bash
# 라즈베리파이에서 실행
mkdir -p /home/pi/autobox/certs

# 인증서 권한 설정
chmod 644 /home/pi/autobox/certs/ca.crt
chmod 644 /home/pi/autobox/certs/client.crt
chmod 600 /home/pi/autobox/certs/client.key
```

### 3.4 방법 A: Mosquitto 브릿지 (권장)

로컬 브로커가 자동으로 EC2로 메시지를 전달합니다.

```bash
# Mosquitto 설치
sudo apt update
sudo apt install -y mosquitto mosquitto-clients

# 브릿지 설정 복사
sudo cp /path/to/mosquitto-bridge.conf /etc/mosquitto/conf.d/bridge.conf

# 설정 수정
sudo nano /etc/mosquitto/conf.d/bridge.conf
```

**수정할 부분:**

```conf
# EC2 브로커 주소 (실제 IP로 변경)
address <EC2_PUBLIC_IP>:8883

# 인증 정보 (실제 비밀번호로 변경)
remote_username raspberry-pi
remote_password <RPI_MQTT_PASSWORD>
```

```bash
# 서비스 재시작
sudo systemctl restart mosquitto
sudo systemctl enable mosquitto

# 로그 확인
sudo journalctl -u mosquitto -f
```

### 3.5 방법 B: Python 클라이언트

직접 EC2에 연결하는 방식입니다.

```bash
# 의존성 설치
pip3 install paho-mqtt python-dotenv

# 환경변수 설정
cd /home/pi/autobox
cp .env.example .env
nano .env
```

**수정할 부분:**

```env
EC2_BROKER_HOST=<EC2_PUBLIC_IP>
EC2_BROKER_PORT=8883
MQTT_USERNAME=raspberry-pi
MQTT_PASSWORD=<RPI_MQTT_PASSWORD>
CA_CERT_PATH=/home/pi/autobox/certs/ca.crt
```

```bash
# 실행
python3 mqtt_client.py
```

## 4. 연결 테스트

### 4.1 EC2 브로커 상태 확인

```bash
# Docker 컨테이너 상태
docker ps

# 브로커 로그
docker logs -f autobox-mqtt

# 메시지 수신 테스트 (EC2에서)
docker exec -it autobox-mqtt mosquitto_sub -h localhost -t "autobox/#" -v
```

### 4.2 라즈베리파이 → EC2 테스트

```bash
# 브릿지 모드: 로컬에 발행 → 자동으로 EC2 전달
mosquitto_pub -h localhost -t "autobox/sensor/data" -m '{"temp": 25.5}'

# 직접 EC2 연결 테스트
mosquitto_pub -h <EC2_IP> -p 8883 \
  --cafile /home/pi/autobox/certs/ca.crt \
  -u raspberry-pi -P <PASSWORD> \
  -t "autobox/test" -m "Hello from RPI"
```

### 4.3 EC2 → 라즈베리파이 명령 테스트

```bash
# EC2에서 명령 발행
docker exec -it autobox-mqtt mosquitto_pub \
  -h localhost -t "autobox/command/start" \
  -m '{"command": "start"}'

# 라즈베리파이에서 수신 확인
mosquitto_sub -h localhost -t "autobox/command/#" -v
```

## 5. 문제 해결

### 5.1 연결 실패

```bash
# EC2 브로커 로그 확인
docker logs autobox-mqtt

# 라즈베리파이 브릿지 로그 확인
sudo journalctl -u mosquitto -f
```

**일반적인 원인:**

| 문제 | 해결 |
|------|------|
| Connection refused | EC2 보안그룹 8883 포트 확인 |
| Authentication failed | 비밀번호 확인 |
| Certificate error | 인증서 경로/권한 확인 |
| Hostname mismatch | `bridge_insecure true` 설정 확인 |

### 5.2 인증서 문제

```bash
# 인증서 확인
openssl x509 -in /home/pi/autobox/certs/ca.crt -text -noout

# EC2 연결 테스트
openssl s_client -connect <EC2_IP>:8883 -CAfile /home/pi/autobox/certs/ca.crt
```

### 5.3 Docker 문제

```bash
# 컨테이너 재시작
docker-compose restart

# 전체 재빌드
docker-compose down
docker-compose up -d --build

# 볼륨 삭제 후 재시작 (주의: 데이터 삭제됨)
docker-compose down -v
docker-compose up -d --build
```

## Topic 구조

```
autobox/
├── sensor/
│   ├── data          # 센서 데이터 (라즈베리파이 → EC2)
│   └── status        # 센서 상태
├── device/
│   ├── status        # 디바이스 상태
│   └── heartbeat     # 연결 유지 신호
├── vehicle/
│   ├── position      # 차량 위치 데이터 (라즈베리파이 → EC2)
│   └── status        # 차량 상태
├── alert/
│   └── notification  # 알림 이벤트
├── camera/
│   └── detection     # 카메라 감지 이벤트
├── command/          # EC2 → 라즈베리파이 명령
│   ├── box-count     # 박스 개수 명령 (프론트 → 라즈베리파이)
│   └── vehicle       # 차량 제어 명령 (start, stop, pause 등)
└── response/         # 라즈베리파이 → EC2 응답 (선택)
    └── box-count     # 박스 개수 명령 응답
```

### 명령 전송 예시 (프론트엔드 → 라즈베리파이)

**박스 개수 전송**
```json
// 토픽: autobox/command/box-count
{
  "command_id": "CMD-A1B2C3D4",
  "command_type": "box_count",
  "box_count": 10,
  "vehicle_id": "AGV-001",
  "priority": "normal",
  "timestamp": "2026-01-26T12:00:00"
}
```

**차량 제어 명령**
```json
// 토픽: autobox/command/vehicle
{
  "command_id": "CMD-E5F6G7H8",
  "command_type": "vehicle_control",
  "command": "start",
  "vehicle_id": "AGV-001",
  "parameters": {},
  "timestamp": "2026-01-26T12:00:00"
}
```
