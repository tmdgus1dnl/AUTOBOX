## 3. 보안그룹 설정 

```bash
# 현재 방화벽 상태 확인
sudo ufw status

# 필요한 포트 열기
sudo ufw allow 22/tcp      # SSH
sudo ufw allow 80/tcp      # Frontend
sudo ufw allow 8000/tcp    # Backend API
sudo ufw allow 8883/tcp    # MQTT TLS

# 방화벽 활성화 (비활성화 상태라면)
sudo ufw enable

# 다시 상태 확인
sudo ufw status numbered
```


## 5. 도커실행
```bash
cd /home/ubuntu/git/S14P11A403/backend
docker-compose up -d --build
docker compose up -d --build
```


## 오류1 : 포트번호 다른거 확인


```bash
# 1883 포트 사용 중인 프로세스 확인
sudo lsof -i :1883

# 또는
sudo netstat -tlnp | grep 1883
```

로컬에 Mosquitto가 이미 실행 중일 가능성이 높습니다:

```bash
# Mosquitto 서비스 중지
sudo systemctl stop mosquitto
sudo systemctl disable mosquitto

# 다시 Docker 실행
docker compose up -d --build
```

만약 다른 프로세스라면:

```bash
# 해당 PID 종료 (lsof에서 확인한 PID로)
sudo kill -9 <PID>
```

## 디버깅
MQTT 송수신 테스트 로그 확인 방법입니다.

---

## 1. Docker 로그 확인

```bash
# MQTT 브로커 로그 (실시간)
docker logs -f autobox-mqtt

# 백엔드 로그 (실시간)
docker logs -f autobox-backend

# 모든 서비스 로그
docker compose logs -f
```

---

## 2. MQTT 메시지 모니터링 (모든 토픽 구독)

```bash
# 컨테이너 안에서 모든 메시지 구독
docker exec -it autobox-mqtt mosquitto_sub -h localhost -p 1883 -t "#" -v
```

특정 토픽만:

```bash
# factory_msg 토픽만
docker exec -it autobox-mqtt mosquitto_sub -h localhost -p 1883 -t "factory_msg/#" -v

# server_msg 토픽만
docker exec -it autobox-mqtt mosquitto_sub -h localhost -p 1883 -t "server_msg/#" -v
```

---

## 3. 테스트 메시지 발행

다른 터미널에서:

```bash
# 테스트 메시지 발행 (라즈베리파이 → 서버)
docker exec -it autobox-mqtt mosquitto_pub -h localhost -p 1883 -t "factory_msg/state" -m '{"status":"running"}'

# 테스트 메시지 발행 (서버 → 라즈베리파이)
docker exec -it autobox-mqtt mosquitto_pub -h localhost -p 1883 -t "server_msg/cmd" -m '{"action":"start"}'
```

---

## 4. 외부에서 TLS로 테스트 (8883 포트)

```bash
# 구독
mosquitto_sub -h <EC2_IP> -p 8883 \
  --cafile mqtt/certs/ca.crt \
  -u backend -P qwert123 \
  -t "#" -v

# 발행
mosquitto_pub -h <EC2_IP> -p 8883 \
  --cafile mqtt/certs/ca.crt \
  -u raspberry-pi -P <비밀번호> \
  -t "factory_msg/state" -m '{"test":"hello"}'
```

## 신규 API 테스트

### 차량 위치 조회
```bash
curl http://localhost:8000/api/v1/vehicle/position
```

### 맵 데이터 조회
```bash
curl http://localhost:8000/api/v1/map/data
```

### 센서 상태 조회
```bash
curl http://localhost:8000/api/v1/sensors/status
```

### 박스 개수 명령 전송 (프론트 → 라즈베리파이)
```bash
curl -X POST http://localhost:8000/api/v1/vehicle/command/box-count \
  -H "Content-Type: application/json" \
  -d '{"box_count": 10, "vehicle_id": "AGV-001"}'
```

### 차량 제어 명령 전송
```bash
# 시작
curl -X POST http://localhost:8000/api/v1/vehicle/command \
  -H "Content-Type: application/json" \
  -d '{"command": "start", "vehicle_id": "AGV-001"}'

# 정지
curl -X POST http://localhost:8000/api/v1/vehicle/command \
  -H "Content-Type: application/json" \
  -d '{"command": "stop", "vehicle_id": "AGV-001"}'

# 긴급 정지
curl -X POST http://localhost:8000/api/v1/vehicle/command \
  -H "Content-Type: application/json" \
  -d '{"command": "emergency_stop", "vehicle_id": "AGV-001"}'
```

## MQTT 명령 수신 확인 (라즈베리파이)

```bash
# 모든 명령 토픽 구독
docker exec -it autobox-mqtt mosquitto_sub -h localhost -t "autobox/command/#" -v
```

예상 출력:
```
autobox/command/box-count {"command_id":"CMD-A1B2C3D4","command_type":"box_count","box_count":10,...}
autobox/command/vehicle {"command_id":"CMD-E5F6G7H8","command_type":"vehicle_control","command":"start",...}
```
