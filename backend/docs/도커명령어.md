## 백엔드 로그 확인

```
docker logs -f autobox-backend
```

환경 변수 설정 미흡: .env 파일에서 MQTT_BROKER_HOST 설정이 주석 처리되어 있었습니다. 이로 인해 백엔드가 mosquitto 컨테이너가 아닌 자기 자신(localhost)에게 연결을 시도하다 실패했습니다.
재연결 로직 부재: 백엔드가 켜질 때 딱 한 번만 연결을 시도하고, 실패하면 포기하도록 되어 있었습니다. 컨테이너 환경에서는 브로커보다 백엔드가 먼저 켜질 수도 있는데, 이를 기다려주는(Retry) 로직이 없어서 계속 '연결 끊김' 상태로 남아있었습니다.
조치 내용:

코드 수정: 백엔드가 시작될 때 5초 간격으로 5번까지 연결을 재시도하도록 코드를 개선했습니다.
설정 수정: .env 파일에서 올바른 호스트(mosquitto)를 바라보도록 주석을 해제했습니다.
컨테이너 재배포: 변경된 설정을 적용하기 위해 컨테이너를 새로 생성했습니다.

## 도커 리빌드

```
# 1. 백엔드 이미지 빌드 및 컨테이너 재시작
cd /home/ubuntu/git/S14P11A403/backend
docker build -t autobox-backend .
docker compose -f docker-compose.yml up -d --build

# 2. 프론트엔드 이미지 빌드 및 컨테이너 재시작
cd /home/ubuntu/git/S14P11A403/frontend
docker build -t autobox-frontend .
docker compose -f docker-compose.yml up -d --build

# 전체 재시작
cd /home/ubuntu/git/S14P11A403/backend
docker compose -f docker-compose.yml down
docker compose -f docker-compose.yml up -d --build
```
