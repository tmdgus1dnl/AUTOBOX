## 2. 민감정보 관리

### 2.1 문제점

현재 `docker-compose.yml`에 실제 데이터베이스 자격 증명이 노출되어 있습니다:

```yaml
# ❌ 현재 상태 (위험)
DATABASE_URL: mysql+pymysql://S14P11A403:실제비밀번호@ssafy-mysql-db.mysql.database.azure.com:3306/S14P11A403
```

### 2.2 해결 방법: 환경 변수 분리

**Step 1: docker-compose.yml 수정**

```yaml
# ✅ 개선된 상태
services:
  backend:
    environment:
      DATABASE_URL: ${DATABASE_URL}
      MQTT_PASSWORD: ${MQTT_PASSWORD}
      CORS_ORIGINS: ${CORS_ORIGINS:-http://localhost}
```

**Step 2: .env 파일 생성 (Git에 포함하지 않음)**

```env
# backend/.env
DATABASE_URL=mysql+pymysql://user:password@host:3306/dbname?charset=utf8mb4
MQTT_PASSWORD=your_secure_password
CORS_ORIGINS=http://localhost,https://your-domain.com
```

**Step 3: .gitignore 확인**

```gitignore
# 이미 포함되어 있음
.env
.env.local
*.env
```

### 2.3 Azure Key Vault 사용 (고급)

프로덕션 환경에서는 Azure Key Vault를 사용하여 비밀을 관리할 수 있습니다:

```python
# app/config.py
from azure.identity import DefaultAzureCredential
from azure.keyvault.secrets import SecretClient

def get_secret_from_keyvault(secret_name: str) -> str:
    credential = DefaultAzureCredential()
    client = SecretClient(
        vault_url="https://your-keyvault.vault.azure.net/",
        credential=credential
    )
    return client.get_secret(secret_name).value
```

## 3. CORS 설정 강화

### 3.1 문제점

현재 모든 도메인에서 API 접근이 허용되어 있습니다:

```python
# ❌ 현재 상태 (위험)
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 3.2 해결 방법

**Step 1: app/main.py 수정**

```python
# ✅ 개선된 상태
from app.config import get_settings

settings = get_settings()

app.add_middleware(
    CORSMiddleware,
    allow_origins=settings.cors_origins_list,  # 특정 도메인만 허용
    allow_credentials=True,
    allow_methods=["GET", "POST", "PUT", "DELETE", "OPTIONS"],
    allow_headers=["Authorization", "Content-Type"],
)
```

**Step 2: 환경 변수 설정**

```env
# 개발 환경
CORS_ORIGINS=http://localhost:5173,http://localhost:80

# 프로덕션 환경
CORS_ORIGINS=https://autobox.example.com,https://api.autobox.example.com
```

## 4. MQTT 보안 강화

### 4.1 문제점

현재 MQTT 브로커가 익명 접속을 허용합니다:

```conf
# ❌ 현재 상태 (위험)
allow_anonymous true
```

### 4.2 해결 방법

**Step 1: mqtt/config/mosquitto.conf 수정**

```conf
# ✅ 개선된 상태
per_listener_settings true

# 내부 통신 (Docker 네트워크 내)
listener 1883 0.0.0.0
allow_anonymous false
password_file /mosquitto/config/passwd

# 외부 통신 (라즈베리파이 연결, TLS 필수)
listener 8883 0.0.0.0
allow_anonymous false
password_file /mosquitto/config/passwd
cafile /mosquitto/certs/ca.crt
certfile /mosquitto/certs/server.crt
keyfile /mosquitto/certs/server.key
tls_version tlsv1.2

# ACL (접근 제어)
acl_file /mosquitto/config/acl
```

**Step 2: 비밀번호 파일 생성**

```bash
cd backend/mqtt/config
./generate-passwd.sh
```

**Step 3: ACL 파일 설정 (mqtt/config/acl)**

```conf
# 백엔드 서버 - 모든 토픽 접근 가능
user backend
topic readwrite autobox/#

# 라즈베리파이 - 센서 데이터 발행, 명령 수신
user raspberry-pi
topic write autobox/sensor/#
topic write autobox/device/#
topic write autobox/vehicle/#
topic read autobox/command/#

# 대시보드 - 읽기 전용
user dashboard
topic read autobox/#
```

## 5. Docker 보안

### 5.1 Non-root 사용자 사용

**backend/Dockerfile 수정:**

```dockerfile
FROM python:3.11-slim

# Non-root 사용자 생성
RUN groupadd -r appgroup && useradd -r -g appgroup appuser

WORKDIR /app

# 의존성 설치 (root로)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 애플리케이션 복사
COPY app/ ./app/

# 소유권 변경
RUN chown -R appuser:appgroup /app

# Non-root 사용자로 전환
USER appuser

EXPOSE 8000
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### 5.2 이미지 취약점 스캔

```bash
# Trivy로 이미지 스캔
docker run --rm -v /var/run/docker.sock:/var/run/docker.sock \
  aquasec/trivy image autobox-backend:latest
```

### 5.3 리소스 제한

```yaml
# docker-compose.yml
services:
  backend:
    deploy:
      resources:
        limits:
          cpus: '1.0'
          memory: 512M
        reservations:
          cpus: '0.5'
          memory: 256M
```

## 6. API 보안

### 6.1 Rate Limiting 적용

**Step 1: 의존성 추가**

```bash
pip install slowapi
```

**Step 2: app/main.py 수정**

```python
from slowapi import Limiter, _rate_limit_exceeded_handler
from slowapi.util import get_remote_address
from slowapi.errors import RateLimitExceeded

limiter = Limiter(key_func=get_remote_address)
app.state.limiter = limiter
app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)

# 라우터에서 사용
@app.get("/api/v1/waybills")
@limiter.limit("100/minute")
async def get_waybills(request: Request):
    ...
```

### 6.2 입력 검증 강화

```python
from pydantic import BaseModel, validator, constr

class WaybillCreate(BaseModel):
    tracking_number: constr(min_length=10, max_length=20, regex=r'^[A-Z0-9]+$')
    
    @validator('tracking_number')
    def validate_tracking_number(cls, v):
        if not v.isalnum():
            raise ValueError('운송장 번호는 영숫자만 허용됩니다')
        return v.upper()
```

### 6.3 보안 헤더 추가

```python
from fastapi.middleware.httpsredirect import HTTPSRedirectMiddleware
from starlette.middleware.trustedhost import TrustedHostMiddleware

# HTTPS 강제 (프로덕션)
# app.add_middleware(HTTPSRedirectMiddleware)

# 허용된 호스트만 접근
app.add_middleware(
    TrustedHostMiddleware,
    allowed_hosts=["autobox.example.com", "localhost"]
)

# 보안 헤더 미들웨어
@app.middleware("http")
async def add_security_headers(request: Request, call_next):
    response = await call_next(request)
    response.headers["X-Content-Type-Options"] = "nosniff"
    response.headers["X-Frame-Options"] = "DENY"
    response.headers["X-XSS-Protection"] = "1; mode=block"
    return response
```

## 7. 데이터베이스 보안

### 7.1 SSL 인증서 검증 활성화

**app/database.py 수정 (프로덕션):**

```python
# ✅ 프로덕션 환경 (인증서 검증 활성화)
connect_args = {}
if "azure" in settings.DATABASE_URL.lower():
    connect_args["ssl"] = {
        "ssl": True,
        "ssl_verify_cert": True,  # 인증서 검증 활성화
        "ssl_verify_identity": True,
        "ssl_ca": "/path/to/DigiCertGlobalRootCA.crt.pem"  # Azure CA 인증서
    }
```

### 7.2 Azure CA 인증서 다운로드

```bash
# Azure MySQL용 CA 인증서 다운로드
curl -o DigiCertGlobalRootCA.crt.pem \
  https://dl.cacerts.digicert.com/DigiCertGlobalRootCA.crt.pem
```

### 7.3 SQL Injection 방지

SQLAlchemy ORM을 사용하면 기본적으로 방지되지만, Raw SQL 사용 시 주의:

```python
# ❌ 위험 (SQL Injection 취약)
db.execute(f"SELECT * FROM users WHERE id = {user_id}")

# ✅ 안전 (파라미터 바인딩)
db.execute(text("SELECT * FROM users WHERE id = :id"), {"id": user_id})
```

## 8. CI/CD 자동화

### 8.1 GitLab CI/CD 파이프라인

**프로젝트 루트에 `.gitlab-ci.yml` 생성:**

```yaml
stages:
  - test
  - build
  - deploy

variables:
  DOCKER_IMAGE: $CI_REGISTRY_IMAGE/backend

# 테스트 단계
test:
  stage: test
  image: python:3.11
  script:
    - cd backend
    - pip install -r requirements.txt
    - pip install pytest pytest-cov
    - pytest --cov=app tests/
  only:
    - merge_requests
    - dev
    - master

# 빌드 단계
build:
  stage: build
  image: docker:latest
  services:
    - docker:dind
  script:
    - docker login -u $CI_REGISTRY_USER -p $CI_REGISTRY_PASSWORD $CI_REGISTRY
    - docker build -t $DOCKER_IMAGE:$CI_COMMIT_SHA -t $DOCKER_IMAGE:latest ./backend
    - docker push $DOCKER_IMAGE:$CI_COMMIT_SHA
    - docker push $DOCKER_IMAGE:latest
  only:
    - dev
    - master

# 배포 단계 (수동)
deploy:
  stage: deploy
  image: alpine:latest
  before_script:
    - apk add --no-cache openssh-client
    - eval $(ssh-agent -s)
    - echo "$SSH_PRIVATE_KEY" | ssh-add -
  script:
    - ssh -o StrictHostKeyChecking=no $DEPLOY_USER@$DEPLOY_HOST "
        cd /app &&
        docker-compose pull &&
        docker-compose up -d --build
      "
  when: manual
  only:
    - master
```

### 8.2 보안 스캔 단계 추가

```yaml
# .gitlab-ci.yml에 추가
security_scan:
  stage: test
  image: aquasec/trivy:latest
  script:
    - trivy fs --exit-code 1 --severity HIGH,CRITICAL ./backend
  allow_failure: true
  only:
    - merge_requests
```
